# Annotate Plan — Iterative Plan Refinement

Process inline annotations in the plan file. Address all notes without implementing. Iterate until the plan is clean.

Inserts between `/create-plan` and `/implement-plan` in the manual workflow:
```
/create-plan → human annotates plan → /annotate-plan → review → repeat if needed → /implement-plan
```

For the pipeline: annotation is human-driven, so this skill applies to the manual workflow only.

---

## Process

### Step 1: Read Plan

1. Read `docs/progress.md` — parse the `**Plan:**` pointer
2. Read the plan file
3. Search for annotation markers: lines starting with `> NOTE:`, `> Q:`, `> CHANGE:`, or `> ???`
4. If no annotations found: output "No annotations found in [plan file]. The plan is clean." and **STOP.**

### Step 2: Process Annotations

For each annotation found:

| Marker | Action |
|--------|--------|
| `> NOTE:` | Informational — acknowledge and incorporate into the plan |
| `> Q:` | Question — answer it inline and update the plan accordingly |
| `> CHANGE:` | Requested change — apply it to the plan |
| `> ???` | Ambiguity — clarify the ambiguous section and update the plan |

Process each annotation:
1. Read the surrounding plan context
2. Address the annotation based on its type
3. Update the plan text to incorporate the resolution
4. Remove the annotation marker line

### Step 3: Update Plan File

1. Write the updated plan file (in place — same file path)
2. Do NOT implement anything — only refine the plan
3. Do NOT modify the prompts file — prompts are regenerated by `/create-plan` if needed

### Step 4: What's Next

```
---

Processed [N] annotations.
  [M] notes acknowledged
  [P] questions answered
  [Q] changes applied
  [R] ambiguities clarified

Review the updated plan: [plan file path]
If more annotations needed: add them and run /annotate-plan again.
If plan is ready: run /implement-plan.

---
```
