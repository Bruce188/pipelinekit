# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.0.1   | :white_check_mark: |
| 0.0.0   | :x:                |

## Reporting a Vulnerability

Please report security vulnerabilities by email to **obruce.shema99@gmail.com**. Use a descriptive subject line such as `pipelinekit security: <short description>`.

Include in the report:

- A description of the issue and the impact you observed.
- Steps to reproduce, or a minimal proof of concept.
- The pipelinekit version (commit SHA or release tag) where you observed the issue.
- Your preferred contact for follow-up.

## Disclosure SLA

The maintainer will acknowledge your report within **7 days** and aim to resolve or publish a coordinated disclosure within **90 days** of the initial report. If a fix is not yet available at the 90-day mark, we will agree a revised timeline with the reporter before any public disclosure.

Please do not file public GitHub issues for security-sensitive reports — use the email address above so the report can be triaged privately before any details become public.
